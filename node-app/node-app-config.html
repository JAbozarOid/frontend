Node App Config

npm init : this command create a package.json file for us

npm init --y : create a package.json file by default config (no questions)

if I add the line "node server.js" to package.json scripts the file server.js will be run

npm i -g yarn : install packages of the project globally on the system, so they are accessable in all projects

npm install loadash install the package loadash in our project

sometimes we need to have some packages in development only not in production : npm i --save-dev typescript -> it goes to devDependencies in package.json file

peerDependencies : if some packages were here means it depend on some other packages to run

React : Library

Next : Framework

with the help of React and Next : we can create single page application (SPA)

React solution : we can update the specific part of a page instead of updating the whole page

Vite : can config a React application project with the all default configuration

Babel : can convert every JavaScript versions we write in out project to the runnable code for browser

JavaScript : is backward compatible(every features are added the old one still works) but it is not

web pack : is a bundler which means all files of html, css and js it packed in a different chunks

different package managers for installing Node packages : npm / yarn / pnpm / bun (installing package)

create command in each package manager : npm create / yarn create / pnpm create ("create command"

npm : has a server which all packages are available on the npm server

yarn : has mirror server on npm server (we download packages from the server of yarn which more powerful)

pnpm : receive packages from npm server and install in our local system(there are caches in our systems)

create a React project with Vite : pnpm create vite

create pnpm globally in our system : npm -i g pnpm

DOM : has a tree structure

DOM Manipulation : after react 16, react create a copy of dom or virtual dom, so in tree just specific detail will be update not whole page

React Structure : we have just one html page, and with the help after react 16 that specific tree will be update

React Structure : instead of apply on the main dom, first create a copy of dom, and then review where is the changes after that just update that change on the main dom

npx : run a batch script (some scripts run in our system)

CRA : create react app pnpm create react-app . ("." means inside of this folder)

In A React Project : there are two important file 1- App.js and 2- index.js

index.js : connect App.js to index.html with document.getElementById('root') in CRA

App.js : every thing we create are in this component in CRA

public folder : is for assets

App.jsx : is in Vite and is equal to App.js in CRA

main.js : is in Vite and is equal to index.js in CRA

There are two ways to create React Components : 1- class component / 2- function component

1- class component :(state full) handle state in react but deprecated after React version 16.0.8

2- functional component : state less

JSX : is understandable html for React

JSX : jsx at compile time convert this "<"div>"Hello world<"/div>" to React compiler at first, like this : React.createElement("div",null,React.createElement("div",null,"Hello World 2"))

Rerender : when state of a component change that component is rerendered (like recomposition)

useState() : is a Hook and this function create a memory for our component

React Props (properties) : uses for two reasons 1- communicate data between components 2- define different state for a component(disable/enable button)
