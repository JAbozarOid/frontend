Node App Config

npm init : this command create a package.json file for us

npm init --y : create a package.json file by default config (no questions)

if I add the line "node server.js" to package.json scripts the file server.js will be run

npm i -g yarn : install packages of the project globally on the system, so they are accessable in all projects

npm install loadash install the package loadash in our project

sometimes we need to have some packages in development only not in production : npm i --save-dev typescript -> it goes to devDependencies in package.json file

peerDependencies : if some packages were here means it depend on some other packages to run

"loadash": "^1.0.0" ^ mean npm goes to get the latest and upper than in 0.0 (minor and path) but we see 0.0
